# Contributing to Approach Map

Thank you for your interest in contributing! Here's how to get started.

## How to Contribute

1. **Fork** the repository and create your branch from `main`.
2. **Make your changes** with clear, focused commits.
3. **Test** your changes thoroughly.
4. **Open a Pull Request** with a clear description of what you changed and why.

## Reporting Bugs

Please open a GitHub Issue with:
- A clear title and description
- Steps to reproduce
- Expected vs. actual behavior
- Any relevant screenshots or logs

## Suggesting Features

Open a GitHub Issue tagged `enhancement` with a description of the feature and the problem it solves.

## Code Style

- Write clear, readable code with comments where helpful.
- Follow existing patterns in the codebase.
- Keep pull requests focused — one change per PR.

## Pull Request Process

1. Ensure your branch is up to date with `main`.
2. Describe your changes in the PR description.
3. A maintainer will review and merge when ready.

## Code of Conduct

Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).
